Netbox Security Refund — Manual Portal
Author Jaivin Prajapati Updated 6 Apr 2026 Status In Review

Overview

A controlled, audited interface for ops leads to manually process Netbox security deposit refunds when the auto-trigger fails or doesn't apply.

Two modules: Wiom Hub sub-tab (ops-facing) and Captive portal (customer-facing, browser-based, for archived customers with no app access).

Problem

When auto-refund fails

  • Device confirmed at partner office via signal — PUT resolved manually, refund not triggered
  • Device at warehouse — PUT closed, but customer never submitted UPI
  • 3PL confirms pickup, Anurag verifies — refund not auto-initiated

No UPI problem

  • Customer never submitted UPI in the Wiom app
  • Once archived, app is locked — no channel to collect UPI
  • Device is with Wiom, refund is owed, but nowhere to send it

Scope — V1

  • Manual refund when customer has submitted UPI
  • Manual refund to original payment source (Juspay)
  • WhatsApp link + captive portal for archived customers to submit UPI
  • Daily limit guardrail + per-customer refund cap
  • Role-gated access
Out of scope V1: active customer UPI submission path (guide customer via app while on call) — separate ticket.

We use Juspay for checkout and don't store instrument details. Ops cannot see card/bank name — only "original payment source". If customer asks, CS informs them the refund goes to the account they paid from. Suggest UPI submission if they want visibility.

Captive Portal

Browser-based flow for archived customers to verify identity and submit UPI. No app required.

Portal ownership (existing vs new) pending alignment with Rajnikant. OTP infra exists and will be reused.

Flow

  1. Ops sends link from Hub → WhatsApp delivered via Clevertap event (dynamic portal link as property)
  2. Customer opens link in browser → sees name + refund amount
  3. OTP verification (delivery channel — SMS or WhatsApp — confirm with Akash)
  4. Customer enters UPI → system resolves bank name
  5. Confirmation screen → customer confirms → refund initiates
  6. Success screen + WhatsApp confirmation sent
